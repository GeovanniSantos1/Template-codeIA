  1. Visão Geral do Produto

1.1 Problema

+-----------------------------------------------------------------------+
| Contexto do Problema                                                  |
|                                                                       |
| O usuário gerencia empréstimos pessoais manualmente através de uma    |
| planilha Excel (.xlsb) com múltiplas abas, VBA, tabelas dinâmicas e   |
| lógicas complexas. Esse processo é frágil: a planilha pode corromper, |
| não é acessível via mobile, não possui controle de acesso, e          |
| dificulta o monitoramento em tempo real de inadimplência, fluxo de    |
| caixa e vencimentos. A gestão de até 30 parcelas por cliente, com     |
| cálculo de juros, multas, WhatsApp e controle de pagamentos, está     |
| sendo feita de forma manual e propensa a erros.                       |
+-----------------------------------------------------------------------+

1.2 Solução

Desenvolver um aplicativo web SaaS (Software as a Service) de gestão de
empréstimos que digitalize e automatize todos os processos atualmente
realizados na planilha, incluindo:

-   Cadastro completo de clientes com dados financeiros e bancários

-   Registro e controle de empréstimos com até 30 parcelas

-   Cálculo automático de juros, multas por atraso e dívida total

-   Dashboard com métricas de fluxo de caixa e inadimplência

-   Alertas de vencimentos e integração com WhatsApp

-   Lançamentos de entradas e saídas (fluxo de caixa)

-   Relatórios e exportação de dados

1.3 Modelo de Negócio

  ---------------- ------------------------------------------------------
  Item             Detalhe

  Modelo           SaaS por assinatura mensal

  Preço MVP        R$ 29,90/mês por usuário

  Pagamento        Integrado com Asaas (gateway de pagamento)

  Autenticação     Clerk (login com e-mail/Google)

  Acesso           Somente usuários com assinatura ativa

                                       MENSAL

  Dívida Atual        Moeda (calc.)    Saldo devedor atual

Regras de Negócio — Empréstimos

-   Parcelas calculadas automaticamente conforme o intervalo
    (diário/semanal/quinzenal/mensal)

-   Suporte a no mínimo 30 parcelas por empréstimo

-   Juros calculados sobre o valor principal

-   Multa calculada automaticamente em cima dos dias de atraso

-   Dívida atual = parcelas restantes + multas acumuladas

-   Um cliente pode ter múltiplos empréstimos ativos simultaneamente

  5. Regras de Negócio

5.1 Cálculos Financeiros

  ---------------------- ------------------------------------------------
  Cálculo                Fórmula / Regra

  Juros ($)              Valor Principal × Juros (%)

  Dívida Total           Valor Principal + Juros ($)

  Valor por Parcela      Dívida Total ÷ Nº Parcelas

  Data das Parcelas      Data empréstimo + (N × intervalo)

  Multa por Atraso       Valor Parcela × (Multa % dia × dias em atraso)

  Dívida Atual           Parcelas não quitadas + Multas acumuladas

  Saldo Credor           Total Recebido − Total Emprestado

  Provisão Total         Prov. Principal + Prov. Juros
  ---------------------- ------------------------------------------------

5.2 Status de Parcelas

-   PENDENTE — parcela gerada e não vencida

-   VENCE HOJE — vencimento = data atual

-   ATRASADA — vencimento anterior à data atual e não paga

-   PAGA — pagamento registrado

-   QUITADA — empréstimo totalmente quitado

5.3 Tipos de Intervalos

-   DIÁRIO — parcelas a cada 1 dia

-   SEMANAL — parcelas a cada 7 dias

-   QUINZENAL — parcelas a cada 15 dias

-   MENSAL — parcelas a cada 30/31 dias (calendário)

5.4 Controle de Acesso

-   Usuário sem assinatura ativa é redirecionado para /billing

-   Middleware Next.js valida autenticação Clerk em todas as rotas
    (protected)

-   Webhook Asaas atualiza status da assinatura em tempo real

-   Dados de cada usuário são isolados (não compartilhados entre contas)

  11. Critérios de Aceite

11.1 Empréstimos

-   Ao criar um empréstimo com N parcelas e intervalo X, o sistema gera
    exatamente N parcelas com datas corretas

-   O valor de cada parcela = Dívida Total ÷ N parcelas

-   Ao marcar uma parcela como paga, o status da parcela muda e a dívida
    atual é recalculada

-   Multa é calculada corretamente como: valor_parcela × (multa_%_dia ×
    dias_de_atraso)

-   Um empréstimo com todas as parcelas pagas muda seu status para
    PAID_OFF

11.2 Dashboard

-   Total Emprestado exibe a soma de todos os empréstimos ativos do
    usuário

-   A lista 'Vence Hoje' mostra somente parcelas com dueDate = data
    atual

-   A lista 'Inadimplentes' mostra somente parcelas com dueDate < data
    atual e status PENDENTE

-   Filtros de mês/ano filtram corretamente entradas e saídas do fluxo
    de caixa

11.3 Autenticação e Assinatura

-   Usuário sem login é redirecionado para /sign-in

-   Usuário com login mas sem assinatura é redirecionado para /billing

-   Após pagamento confirmado pelo Asaas, o usuário tem acesso imediato

-   Dados de um usuário não são acessíveis a outros usuários

11.4 WhatsApp

-   Ao clicar 'Enviar cobrança', abre o WhatsApp com o número e mensagem
    corretos pré-preenchidos

-   A mensagem inclui o nome do cliente, valor da parcela e data de
    vencimento

-   A contagem de caracteres é exibida e atualiza em tempo real
